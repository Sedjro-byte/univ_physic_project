# Architecture du projet "Ballistic"

* Le dossier principal contient les fichiers principaux, qui commencent par "main"
pour mener le projet dans ces différentes phases
* Le dossier **Ballistic** contient tous les fichiers contenant les classes du projet.
* Le nom du fichier contenant une classe est celui de la classe en lettres minuscules.
* Chaque classe est testée dans un fichier principal dont le nom démarre par **main**
* Les noms sont fixés, et identiques pour tous.
* Il suffit donc de remplir les fichiers ou les parties manquantes.

La décomposition proposée crée beaucoup de fichiers, l'avantage est qu'on peut 
faire fonctionner les morceaux les uns indépendamment des autres, ce qui sera
plus facile à corriger.
